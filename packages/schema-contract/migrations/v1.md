# FoodMemory v1 Migration Notes

FoodMemory v1 is the initial FoodLife schema contract.

## Current Status

- `schemaVersion` is required and must be `1`.
- No migration from an earlier FoodMemory schema exists because this is the first version.
- Platform repositories must validate against this contract before storing or reading records.

## Future Migration Rule

Future schema changes must update the Zod validator, portable JSON Schema artifact, fixtures, conformance expected outputs, and contract tests before platform repository or UI code changes.

Optional fields are represented by omission. `null` values are not valid unless a future schema version deliberately adds and documents them.
